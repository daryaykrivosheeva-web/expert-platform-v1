# AI Product Studio — Expert Platform (услуги и клиенты)

Этот репозиторий — рабочее пространство студии по услугам, которые она оказывает клиентам прямо сейчас.
Пять папок, каждая — своя область:

| Папка | Что внутри |
|---|---|
| [`Сайт/`](./Сайт/) | Шаблон сайта Expert Platform v1 (Next.js) — код, конфигурация, документация. Берётся за основу для сайта каждого нового клиента. |
| [`СРМ/`](./СРМ/) | Ссылка на CRM-систему (Notion Expert Workspace) и инструкция для клиента, как её подключить. |
| [`Онлайн-запись/`](./Онлайн-запись/) | Сервис онлайн-записи — пока не подключён, документ-заглушка. |
| [`Клиенты/`](./Клиенты/) | Подпапка на каждого клиента: материалы, полученные от клиента (бриф, фото), и краткое summary того, что для него сделано. |
| [`Портфолио/`](./Портфолио/) | Подпапка на каждого клиента: скриншоты и видео готового сайта — для рекламы (Авито, Kwork) и презентации студии. |

## Как это устроено

- **`Сайт/`** — это отдельный, самодостаточный Next.js-проект (свой `package.json`, деплоится отдельно).
  При деплое на Vercel/Netlify указывайте **Root Directory = `Сайт`**.
- **`Клиенты/<Имя>/`** и **`Портфолио/<Имя>/`** — по одной папке на клиента в каждой; не смешивайте с кодом сайта.
- Полный коммерческий трек по клиенту (бриф, статус проекта, задачи, история) ведётся в отдельном
  студийном репозитории `AI-Product-Studio` → `13_Clients/CLIENT-XXX_Имя/`. Здесь, в `Клиенты/`, —
  только исходные материалы от клиента и краткая справка, что сделано и где это искать в `Сайт/`.

## Новый клиент — что делать

1. Создать конфиг ниши/клиента: `Сайт/src/config/examples/<ниша>.config.ts`.
2. Подключить его в `Сайт/src/config/site.config.ts`.
3. Завести `Клиенты/<Имя>/` с материалами от клиента и кратким summary.
4. После публикации — собрать `Портфолио/<Имя>/` (скриншоты + видео).
5. Завести полную карточку клиента в `AI-Product-Studio/13_Clients/CLIENT-XXX_Имя/`.

Подробности по адаптации шаблона — в `Сайт/builder/README.md`.
