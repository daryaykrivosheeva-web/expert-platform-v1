# EXAMPLES

Данный документ содержит примеры работы Website Product Builder.

---

# Пример 1. Новый клиент, новая ниша

Пользователь:

Создай адаптацию для клиента Мария Полякова (психолог).

Действия:

1. Создать `src/config/examples/psychologist.config.ts` на основе типа `SiteConfig`.
2. Не изменять `src/components/` и `src/app/`.
3. Создать `../Клиенты/<Клиент>/` с материалами клиента (бриф, фото) и кратким summary.
4. Завести полную карточку клиента в `AI-Product-Studio/13_Clients/CLIENT-XXX_Имя/` (студийный репозиторий, коммерческий/организационный трек).

---

# Пример 2. Получены материалы клиента

Пользователь:

Вот фотографии клиента.

Действия:

Поместить фотографии в

`public/images/` (оптимизировать через sharp: max ~2000px, ~200KB)

и указать пути в конфиге (`hero.image`, `about.photo`, `expert.photo`).

---

# Пример 3. Заполнение данных клиента

Пользователь:

Вот услуги, стоимость и контакты.

Действия:

Заполнить соответствующие поля в `src/config/examples/<niche>.config.ts` (`services`, `contacts`).

Не изменять структуру типа `SiteConfig`.

---

# Пример 4. Проверка проекта

Пользователь:

Проверь проект.

Website Product Builder обязан проверить:

✓ конфиг клиента (`<niche>.config.ts`)

✓ фото в `public/images/`

✓ контакты

✓ услуги и стоимость

✓ SEO

✓ секцию `legal` (реквизиты, тип регистрации, дата)

Если чего-либо не хватает — вывести список недостающих материалов.

Не продолжать работу до получения информации.

---

# Пример 5. Создание сайта под клиента

Пользователь:

Создай сайт.

Website Product Builder обязан:

1. Использовать только шаблон Expert Platform (`src/`).
2. Использовать только конфиг клиента и `public/images/`.
3. Не изменять архитектуру сайта.
4. Заполнить в конфиге: тексты, фотографии, логотип, цвета, контакты, услуги, стоимость, SEO, `legal`.
5. Юридические документы генерируются автоматически из `legal` — вручную не создаются.
6. Указать `site.config.ts` → реэкспорт конфига клиента.
7. Проверить сборку (`npm run build`, `npm run typecheck`).
8. Подготовить сайт к публикации.

---

# Пример 6. Если клиент изменил стоимость

Пользователь:

Клиент изменил стоимость консультации.

Действия:

Изменить `services` в `src/config/examples/<niche>.config.ts`.

После этого пересобрать сайт (`npm run build`).

Архитектуру сайта не изменять.

---

# Пример 7. Если клиент прислал новые фотографии

Пользователь:

Замени фотографии.

Действия:

Обновить изображения в `public/images/` (пройти через sharp, сохранить те же имена файлов или обновить пути в конфиге).

После этого пересобрать сайт.

---

# Пример 8. Передача проекта

После завершения адаптации Website Product Builder обязан:

✓ Проверить сайт

✓ Проверить ссылки

✓ Проверить формы (чекбокс согласия перед Telegram/WhatsApp)

✓ Проверить юридические документы (`/legal/*`)

✓ Проверить адаптивность

✓ Подготовить скриншоты и видео в `../Портфолио/<Клиент>/`

✓ Обновить summary в `../Клиенты/<Клиент>/` и карточку в `AI-Product-Studio/13_Clients/CLIENT-XXX_Имя/`

После этого сообщить пользователю о готовности проекта (GitHub / Demo / Production).

# Пример 9. Полностью автоматическая сборка проекта

Пользователь:

Собери проект полностью.

Website Product Builder обязан:

1. Проверить конфиг клиента (`<niche>.config.ts`).
2. Если данных не хватает — вывести список недостающей информации.
3. Если данных достаточно:
   - собрать сайт;
   - убедиться, что юридические документы сгенерировались из `legal`;
   - подключить конфиг через `site.config.ts`;
   - подготовить проект к публикации.
4. Предоставить итоговый отчет о выполненных действиях.
