# EXAMPLES

Данный документ содержит примеры работы Website Product Builder.

---

# Пример 1. Новый клиент

Пользователь:

Создай новый проект для клиента Мария Полякова (психолог).

Действия:

1. Создать копию PROJECT_TEMPLATE.
2. Назвать папку:

PROJECT-001_Maria_Polyakova

3. Не изменять PROJECT_TEMPLATE.

---

# Пример 2. Получены материалы клиента

Пользователь:

Вот фотографии клиента.

Действия:

Поместить фотографии в

01_Source/Images/

---

Пользователь:

Вот логотип.

Действия:

Поместить логотип в

01_Source/Logo/

---

Пользователь:

Вот брендбук.

Действия:

Поместить его в

01_Source/Brand/

---

Пользователь:

Вот документы клиента.

Действия:

Поместить их в

01_Source/Documents/

---

# Пример 3. Заполнение данных клиента

Пользователь:

Вот услуги, стоимость и контакты.

Действия:

Заполнить

CLIENT_DATA.yaml

Не изменять структуру файла.

---

# Пример 4. Проверка проекта

Пользователь:

Проверь проект.

Website Product Builder обязан проверить:

✓ CLIENT_DATA.yaml

✓ Фото

✓ Логотип

✓ Контакты

✓ Услуги

✓ Стоимость

✓ SEO

✓ Юридические данные

Если чего-либо не хватает — вывести список недостающих материалов.

Не продолжать работу до получения информации.

---

# Пример 5. Создание сайта

Пользователь:

Создай сайт.

Website Product Builder обязан:

1. Использовать только шаблон Expert Platform.

2. Использовать только материалы из

01_Source/

3. Не изменять архитектуру сайта.

4. Изменить:

- тексты
- фотографии
- логотип
- цвета
- контакты
- услуги
- стоимость
- SEO

5. Создать юридические документы.

6. Разместить ссылки в футере.

7. Проверить сборку.

8. Подготовить сайт к публикации.

---

# Пример 6. Если клиент изменил стоимость

Пользователь:

Клиент изменил стоимость консультации.

Действия:

Изменить данные в

CLIENT_DATA.yaml

После этого обновить сайт.

Архитектуру сайта не изменять.

---

# Пример 7. Если клиент прислал новые фотографии

Пользователь:

Замени фотографии.

Действия:

Обновить изображения в

01_Source/Images/

После этого пересобрать сайт.

---

# Пример 8. Передача проекта

После завершения проекта Website Product Builder обязан:

✓ Проверить сайт

✓ Проверить ссылки

✓ Проверить формы

✓ Проверить юридические документы

✓ Проверить адаптивность

✓ Подготовить материалы в

07_Delivery/

После этого сообщить пользователю о готовности проекта.

# Пример 9. Полностью автоматическая сборка проекта

Пользователь:

Собери проект полностью.

Website Product Builder обязан:

1. Проверить папку 01_Source.
2. Проверить CLIENT_DATA.yaml.
3. Если данных не хватает — вывести список недостающей информации.
4. Если данных достаточно:
   - собрать сайт;
   - создать юридические документы;
   - обновить 02_Website;
   - обновить 07_Delivery;
   - подготовить проект к публикации.
5. Предоставить итоговый отчет о выполненных действиях.
