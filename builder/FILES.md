# FILES

Данный документ описывает, какие файлы Website Product Builder использует в работе.

---

# Читает / Заполняет

`src/config/examples/<niche>.config.ts` — весь контент, цены, контакты, брендинг, SEO, `legal`.

`src/types/config.ts` — контракт `SiteConfig` (не менять при адаптации под клиента).

---

# Изменяет

`src/config/examples/<niche>.config.ts` (создает новый файл для нового клиента/ниши)

`src/config/site.config.ts` (реэкспорт активного клиента)

`public/images/` (фотографии клиента)

---

# Создает

`portfolio/<niche>/<client>/` — скриншоты и видео готовой адаптации для портфолио.

---

# Не имеет права изменять

`src/app/`

`src/components/`

`src/lib/`

`src/legal/`

архитектуру сайта и структуру шаблона.

---

# Главный источник данных

`src/config/examples/<niche>.config.ts`

---

# Если данных недостаточно

Website Product Builder обязан:

- остановить выполнение;
- сформировать список недостающей информации;
- дождаться пользователя.

Запрещено придумывать данные самостоятельно.
